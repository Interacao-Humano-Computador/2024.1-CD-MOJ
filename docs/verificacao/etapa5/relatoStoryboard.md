## <a>Introdução</a>

Neste artefato, nosso objetivo é relatar de forma detalhada a verificação do artefato <a href="https://interacao-humano-computador.github.io/2024.1-SIGAA/#/DesignAvalia%C3%A7%C3%A3oDesenvolvimento/N%C3%ADvel1/Storyboard/RelatoResultadosStoryboard">Relato dos Resultados dos Storyboards</a> desenvolvido pelo <a href="https://interacao-humano-computador.github.io/2024.1-SIGAA/#/">grupo 02</a>. Essa verificação é fundamental para garantir que todas as atividades planejadas estejam devidamente alinhadas com os objetivos do projeto, os prazos estabelecidos e as expectativas dos stakeholders. Analisaremos a coerência das datas, a distribuição das tarefas e a viabilidade dos Princípios Gerais do Projeto, assegurando que eles sejam realistas e exequíveis.

## <a>Checklist</a>

O relato dos resultados do storyboard é uma etapa fundamental na avaliação de interfaces e experiências do usuário, documentando os achados obtidos durante a análise. Este relato deve incluir os objetivos e o escopo da avaliação, o método de avaliação empregado, o número e o perfil dos participantes, um sumário dos dados coletados, a interpretação e análise desses dados, uma lista dos problemas encontrados e um planejamento para o reprojeto do sistema. Para garantir que todos esses aspectos sejam adequadamente verificados, foi elaborado um checklist. A seguir, apresentamos o checklist detalhado na Tabela 1 referente a verificação dos resultados consolidados.

<font size="3"><p style="text-align: center"><b>Tabela 1</b> - Verificação dos Relatos dos Resultados dos Storyboards.</p></font>
